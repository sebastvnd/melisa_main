Custom HTML Files Served by MNode

📋 How It Works

MNode = Worker Node

  • MNode menjalankan backend service Anda
  • Serve static files (HTML, CSS, JS) dari folder public/html
  • Provide API endpoints untuk data/operations
  • Auto-register ke Melisa Management API saat startup
  • Proxy Melisa akan route traffic ke MNode sesuai routing rules

📁 File Structure

mnode/
├── mnode.conf          ← Configuration file
├── public/
│   └── html/           ← Your static files go here
│       ├── index.html  ← Homepage
│       ├── style.css   ← Stylesheets
│       ├── app.js      ← JavaScript
│       └── ...
└── src/
    ├── main.rs
    ├── config.rs       ← Reads mnode.conf
    ├── handler.rs      ← Serves files from public/html
    └── ...

⚙️ Configuration (mnode.conf)

[static_files]
# Directory untuk HTML/CSS/JS files (relatif dari mnode root)
directory = "./public/html"
enabled = true

[registration]
# Auto-register ke Melisa
melisa_host = "127.0.0.1"
melisa_port = 8888
node_domain = "api.example.com"
node_route_path = "/mnode"

📝 Steps to Use Custom HTML

  1. Edit mnode.conf untuk konfigurasi
  2. Create/edit HTML files di public/html/
    • index.html = Homepage (served at /)
    • Other files served dengan relative path
  3. Add CSS/JS/images di folder yang sama
  4. Run: cargo run --bin mnode
  5. Access via: http://localhost:3000/

🔗 Available Endpoints

  • GET / → Serve index.html dari public/html
  • GET /page.html → Serve public/html/page.html
  • GET /style.css → Serve public/html/style.css
  • GET /api/info → Node information (JSON)
  • GET /api/health → Health check (JSON)

🔒 Security Notes

  • Directory traversal protected (.. paths blocked)
  • Only files served, not directories
  • MIME types auto-detected
  • Static files cacheable (1 hour max-age)

🧪 Test Your Setup

# Terminal 1 - Start Melisa
cargo run --bin melisa_beta

# Terminal 2 - Start MNode
cd mnode && cargo run --bin mnode

# Terminal 3 - Test endpoints
curl http://localhost:3000/                  # index.html
curl http://localhost:3000/api/info          # JSON
curl http://localhost:3000/api/health        # Health check

# Check Melisa management
curl http://localhost:8888/nodes             # List registered nodes
